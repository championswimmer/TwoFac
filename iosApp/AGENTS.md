# iosApp AGENTS.md

`iosApp` is the Xcode workspace containing the native iOS wrapper and the Apple Watch companion targets.

## What this module does
It provides the native Swift/Objective-C scaffolding required to boot the Kotlin multiplatform code on Apple devices. It initializes the `ComposeUIViewController` for the main iOS app. Watch sync is driven entirely by `IosCompanionSyncCoordinator` (Kotlin, in `composeApp/src/iosMain`), which pushes snapshots over `WCSession`. The watchOS side decodes and displays OTP codes in pure SwiftUI with shared timer/theme semantics bridged from Kotlin.

## Dependencies
- **iOS App**: Depends on the `TwoFacUIKit` framework generated by `:composeApp`.
- **watchOS App**: Depends directly on the `TwoFacKit` framework generated by `:sharedLib`.

## Platforms
- **iOS**
- **watchOS**

## Libraries Used
- [SwiftUI](https://developer.apple.com/xcode/swiftui/) - Native UI framework used heavily in the watchOS extension.
- [WatchConnectivity](https://developer.apple.com/documentation/watchconnectivity) - Apple's framework for communicating state between an iOS app and a watchOS companion.
- Core Foundation / UIKit - Standard native iOS bootstraps.

## Code Structure
- `iosApp/`: The iOS application target.
  - `ContentView.swift`: Embeds the Compose UI.
  - *(Watch sync is handled by `IosCompanionSyncCoordinator` in `composeApp/src/iosMain`; no standalone Swift sync coordinator exists.)*
- `watchApp/`: The watchOS shell target containing watch app resources and entitlements.
- `watchAppExtension/`: The watchOS SwiftUI target and runtime logic.
  - `TwoFacWatchApp.swift`: SwiftUI app entrypoint for the watch experience.
  - `WatchConnectivityManager.swift`: Listens for incoming snapshots, decodes them using `TwoFacKit`, and persists them locally.
  - `WatchThemeTokens.swift`: Bridges `TwoFacKit` theme/timer tokens into native SwiftUI `Color` values.
  - `WatchExtensionContentView.swift`: A SwiftUI pager of accounts with countdown bars colored via shared timer semantics.
  - `IssuerBrandIconView.swift`: Renders issuer brand icons from shared glyphs using the bundled Font Awesome Brands font.
    - Uses `Font.custom("FontAwesome6Brands-Regular", size: …)` with glyph lookup from `IssuerIconCatalog.shared.glyphForIconKey(iconKey:)` (from `TwoFacKit`).
    - Requires `iosApp/watchApp/Fonts/fa_brands_400_regular.ttf` to be bundled by the `watchApp` target and registered under `UIAppFonts` in `watchApp/Info.plist`.
    - Unknown or unavailable glyphs fall back to a code-drawn circled `?` placeholder.
