# iosApp AGENTS.md

`iosApp` is the Xcode workspace containing the iOS application wrapper and the Apple Watch companion app.

## What this module does
It provides the native Swift/Objective-C scaffolding required to boot the Kotlin multiplatform code on Apple devices. It initializes the `ComposeUIViewController` for the main iOS app and natively handles the `WatchConnectivity` layer to push secrets. For the Apple Watch, it uses pure SwiftUI to display the OTP codes without the heavy footprint of Compose Multiplatform.

## Dependencies
- **iOS App**: Depends on the `TwoFacUIKit` framework generated by `:composeApp`.
- **watchOS App**: Depends directly on the `TwoFacKit` framework generated by `:sharedLib`.

## Platforms
- **iOS**
- **watchOS**

## Libraries Used
- [SwiftUI](https://developer.apple.com/xcode/swiftui/) - Native UI framework used heavily in the watchOS extension.
- [WatchConnectivity](https://developer.apple.com/documentation/watchconnectivity) - Apple's framework for communicating state between an iOS app and a watchOS companion.
- Core Foundation / UIKit - Standard native iOS bootstraps.

## Code Structure
- `iosApp/`: The iOS application target.
  - `ContentView.swift`: Embeds the Compose UI.
  - `WatchSyncCoordinator.swift`: Responsible for pushing the encrypted account snapshot over `WCSession` to the watch.
- `watchApp/` & `watchAppExtension/`: The watchOS targets.
  - `WatchConnectivityManager.swift`: Listens for incoming snapshots, decodes them using `TwoFacKit`, and persists them locally.
  - `WatchExtensionContentView.swift`: A SwiftUI view displaying a vertical pager of accounts with native countdown arcs.
